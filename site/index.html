Tally
Install Usage Why

An over-optimized cloc clone.

Tally counts files, lines, blanks, comments, and code. It is basically the thing you already know, but written in Rust and tuned because that was more fun than leaving it alone.

There is no platform here. No dashboard. No insight engine. It walks a tree, guesses the language, prints a table, and stops. 

$ tally .
Language Files Lines Blank Comment Code
-------- ----- ----- ----- ------- ----
Rust         6  1235   184       4 1047
TOML         9   972   146       2  824
Text         1   200    73       0  127
-------- ----- ----- ----- ------- ----
Total       16  2407   403       6 1998

Install it if this is your kind of problem.

The installer downloads a prebuilt binary for macOS or Linux, makes it executable, and puts it in ~/.local/bin. 

curl -fsSL https://jafupy.com/tally.sh | sh
tally .

It has about three ideas.

By default, Tally respects git ignore rules, uses a small worker pool for directories, and prints one table. The flags are there for the boring cases:

  • --all includes files that gitignore rules would normally hide.
  • --threads controls worker count for larger trees.
  • --verbose prints extra diagnostics, including unknown file formats.

Why bother?

Because sometimes you want the shape of a repo without waiting, installing a larger tool, or pretending that a line-count table is strategic analysis.

Tally is for terminal people with a mild performance problem. If cloc is already good enough for you, keep using it. That is a reasonable position.

GPL-3.0-or-later. Counts lines. Tries very hard.
